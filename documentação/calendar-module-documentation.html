Módulo Calendário SaaS - Documentação Técnica

1. Visão Geral

O módulo de calendário foi desenvolvido para gerenciar atividades multitenant integradas diretamente à tabela activities do Supabase. Ele oferece visualizações flexíveis e um CRUD completo com isolamento de dados por organização.

Supabase Auth/DB React Hook Architecture Multi-tenant (RLS Ready)

3. Fluxo de Dados e Multi-tenant

O sistema garante o isolamento de dados em todas as camadas:

  • Service: Todas as queries (Select, Insert, Update, Delete) incluem a cláusula .eq('organization_id', organizationId).
  • Hook: Utiliza o AuthContext para injetar automaticamente o ID da organização ativa em todas as chamadas.
  • Interface: Os eventos são filtrados preventivamente por data e organização.

Funcionalidades Principais

  • Multi-tenant: Filtro automático por organization_id via row-level security e filtros de aplicação.
  • Visualização Tripla: Alternância fluida entre visões Mensal, Semanal e Diária.
  • Design Premium: Estética moderna com sombras suaves, bordas arredondadas (32px) e animações de entrada.
  • Gestão de Eventos: Modal interativo com botões visuais para seleção de tipo de atividade (Chamada, WhatsApp, Reunião, Nota).
  • Integração de Leads: Associação opcional de atividades a leads existentes no sistema.

Visualizações

O módulo suporta três modos principais de visualização:

Mês: Grade clássica 7x5 para visão geral de longo prazo.
Semana: Colunas verticais para planejamento detalhado da semana atual.
Dia: Foco total nas atividades do dia selecionado.

4. Integração Google Calendar (Roteiro)

A estrutura está preparada no arquivo googleCalendar.service.ts. Para ativar:

  1. Configurar projeto no Google Cloud Console e obter ClientID/API Key.
  2. Implementar o fluxo OAuth2 no connectGoogleCalendar.
  3. Mapear os campos da Tabela activities para o recurso Event do Google API no helper syncEventToGoogle.
  4. Adicionar um campo google_event_id na tabela para manter o rastreamento.

5. Boas Práticas e Expansão

  • Realtime: Pode ser adicionado usando supabase.channel('activities').on(...) dentro do hook.
  • Drag & Drop: A estrutura do EventCard e renderMonthView está preparada para bibliotecas como dnd-kit ou react-beautiful-dnd.
  • Performance: O carregamento é otimizado por mês, reduzindo o tráfego de dados.
